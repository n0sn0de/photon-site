Token Mechanics

How Photon works

A one-way burn mechanism ensures PHOTON's supply is forever tied to ATONE's sacrifice.

click to replay

🔢 The Math

conversion_rate = (1,000,000,000 − photon_supply) / atone_supply

As more PHOTON is minted, the rate decreases. As ATONE inflates (7-20% annually), the rate also decreases. Early burners get more PHOTON per ATONE.

🚫 No Going Back

PHOTON cannot be converted back to ATONE at the protocol level. This one-way gate ensures the staking token's security properties are never undermined by fee-token holders.

📊 Supply Dynamics

Max supply: 1 billion PHOTON. This cap can never be reached because it would require burning all ATONE — impossible while validators stake. PHOTON is non-inflationary by design.

⚙️ Fee Transition

Currently both ATONE and PHOTON are accepted for fees (transition period). A future governance vote will make PHOTON the exclusive fee token across all shards.

The dual-token thesis

Separating staking from fees isn't just elegant — it's a security imperative for IBC hub chains.

Why this matters for security

When a single token handles both staking and fees, attackers can buy the token on exchanges, stake enough to control 2/3 of consensus, and manipulate IBC transactions to steal pegged assets. Even with slashing, the stolen assets may exceed the slashing penalty.

With separated tokens, accumulating fee tokens gives zero consensus power. Attacking requires acquiring ATONE — which is illiquid by design, has no LSD derivatives, and is locked in staking. The economic barrier to attack becomes prohibitively high.

Origin

Back to the beginning

The dual-token model wasn't invented for AtomOne. It was the original Cosmos vision from 2019 — a multi-token architecture that separated staking from payments.

Over time, the Cosmos Hub drifted. ATOM became treated as a currency, a store of value, a DeFi building block. Liquid staking derivatives let people earn staking rewards without actually participating in security. Proposal #848 reduced inflation to make ATOM "more valuable" as money — undermining its purpose as a security mechanism.

Jae Kwon and the ATOM 1.0 community said: enough. They forked Gaia to create AtomOne — not as competition, but as a return to first principles. PHOTON is the fee token that ATOM was always supposed to have.

In May 2025, Proposal #8 (the v2 upgrade) introduced PHOTON with 99.97% community approval. The original vision, finally realized.

How the arbitrage works

When minting is cheaper than buying, arbitrageurs can burn ATONE → mint PHOTON → sell on Osmosis for profit. This increases PHOTON supply and DEX selling pressure, pushing the market price down toward the mint cost.

When buying is cheaper than minting, traders buy PHOTON on Osmosis instead of burning ATONE. This reduces minting activity, slowing supply growth and letting the market find equilibrium.

Over time, these forces keep the market price loosely anchored to the mint cost — though they can diverge based on demand, speculation, and liquidity.

Context: Ethereum's fee burn

Ethereum's EIP-1559 burns ~1-3% of ETH supply annually at peak activity. AtomOne's fixed-cap PHOTON means fee burns permanently reduce circulating supply with no new issuance to offset — making even modest usage levels deflationary for PHOTON holders.

Under the Hood

The actual code

From x/photon — the Go module that runs it all. No magic, just math.

keeper.go Conversion Rate Function 
// PhotonConversionRate returns the conversion rate for
// converting bond denom to photon.
// NOTE: bondDenomSupply cannot be zero when the chain is
// producing blocks — validators must have staked bond denom.
func (k Keeper) PhotonConversionRate(
    _ context.Context,
    bondDenomSupply, uphotonSupply math.LegacyDec,
) math.LegacyDec {
    remainMintable := math.LegacyNewDec(MaxSupply).Sub(uphotonSupply)
    if remainMintable.IsNegative() {
        return math.LegacyZeroDec()
    }
    return remainMintable.Quo(bondDenomSupply)
}

The formula is beautifully simple: rate = (1B − photon_supply) / atone_supply

As PHOTON supply grows toward 1B, the numerator shrinks → rate decreases. As ATONE inflates (7-20% annually), the denominator grows → rate decreases. Early minters get more PHOTON per ATONE burned.

msg_server.go Mint Flow (4 steps) 
// MintPhoton implements the MsgServer.MintPhoton method.
func (k msgServer) MintPhoton(goCtx context.Context,
    msg *types.MsgMintPhoton,
) (*types.MsgMintPhotonResponse, error) {

    // Compute photons to mint using current rate
    conversionRate := k.PhotonConversionRate(ctx,
        bondDenomSupply, uphotonSupply)
    uphotonToMint := bondDenomToBurn.Amount.
        ToLegacyDec().Mul(conversionRate).TruncateInt()

    // The atomic 4-step swap:
    // 1) Send ATONE from user → photon module
    k.bankKeeper.SendCoinsFromAccountToModule(...)

    // 2) Burn the ATONE (permanently destroyed)
    k.bankKeeper.BurnCoins(ctx, types.ModuleName, coinsToBurn)

    // 3) Mint fresh PHOTON into module
    k.bankKeeper.MintCoins(ctx, types.ModuleName, coinsToMint)

    // 4) Send PHOTON from module → user
    k.bankKeeper.SendCoinsFromModuleToAccount(...)
}

All four steps happen atomically in a single transaction. If any step fails, the entire operation reverts — your ATONE is safe. The MsgMintPhoton transaction type is fee-exempt by default, so you don't need PHOTON to mint your first PHOTON.

params.go Module Parameters 
const (
    defaultMintDisabled = false

Two key governance-controlled parameters: MintDisabled can pause minting via governance vote, and TxFeeExceptions lists message types that don't require fee payment. Currently only MsgMintPhoton is exempt — a clever bootstrap mechanism.

Constitutional Law

Written in the constitution

PHOTON isn't a feature — it's a constitutional mandate. Article 3, Section 5 of the AtomOne Constitution.

Article 3 — Economic Model Section 5: The PHOTON Token

The PHOTON shall be the only fee token except for ATONE to PHOTON burn transactions. This applies for all transactions on the root and core shards, and all IBC and ICS payments.

ATONE tokens may be burned to PHOTON tokens at a conversion rate set by law such that the total amount of PHOTONs mintable through burning ATONE tokens shall be capped at 1B PHOTON tokens.

PHOTONs cannot be converted back into ATONE tokens.

Fee Economics

Dynamic fee pricing

AtomOne uses an adaptive EIP-1559 mechanism to automatically adjust transaction fees based on network congestion.

📈

When blocks are full

Base gas price increases multiplicatively — fees spike to discourage spam and prioritize legitimate transactions. The learning rate scales aggressively during sustained congestion.

📉

When blocks are empty

Base gas price decreases additively — fees gradually return to the floor. This asymmetry (fast up, slow down) prevents fee oscillation and maintains stability.

🛡️

DOS protection

Born from a real March 2025 attack where bloated multi-send transactions overloaded nodes. The dynamic fee module automates the response that previously required manual validator intervention.

Approved via Proposal #15, the x/dynamicfee module implements AIMD (Additive Increase Multiplicative Decrease) EIP-1559 — a refinement of Ethereum's fee model with an adaptive learning rate that responds faster to demand spikes. Since PHOTON is the fee token, dynamic pricing directly affects PHOTON's utility value.

Decentralization Incentive

The Nakamoto bonus

A novel reward mechanism that incentivizes delegators to spread stake across smaller validators — improving the chain's Nakamoto coefficient.

How it works

Block rewards are split into two components:

  • Proportional Reward — distributed based on each validator's stake (the traditional model)
  • Nakamoto Bonus — distributed uniformly across all active validators, regardless of stake size

The total reward stays the same — but smaller validators get a disproportionately larger share, making delegation to them more attractive. Delegators who spread their stake earn higher effective APR.
